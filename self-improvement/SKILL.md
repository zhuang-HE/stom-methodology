# Self-Improvement - Agent 自我优化

## 🎯 定位

系统性反思当前会话和历史记忆 → 识别执行盲点、低效模式、改进机会 → 持久化到工作记忆。

> **详细参考**（意图预判完整流程、质量评分系统、报告模板、自动触发决策树、Skill 创建 MVP 流程、跨会话模式检测引擎）→ `references/detailed-flows-and-templates.md`

---

## 🧠 五个反思维度

| 维度 | 核心问题 |
|------|---------|
| **D1 执行质量** | 多次重试？工具选择最优？冗余步骤？重复错误？ |
| **D2 用户理解** | 新偏好？说的和想要的一致？哪里误解？ |
| **D3 知识漏洞** | 应封装为 Skill？触发词盲区？踩坑该记录？ |
| **D4 记忆系统** | 实质性工作已写入？需更新？需整理？ |
| **D5 序列模式**（v3.0 新增） | 跨 turn 模式：低效循环？工具惯性？重复任务？ |

---

## 📋 工作流（5 步）

### Step 0 · 意图预判
快速扫描 Skill 列表 → 有匹配则预加载，无匹配进入 Step 1。反模式：简单任务不预判重型 Skill。

### Step 1 · 读取上下文
读 MEMORY.md（长期）+ 今日日志 + 回顾当前会话。

### Step 2 · 系统反思
按 D1-D5 维度反思：✅ 做得好 / 🔧 做得不够 / 📚 新知识 / 🚀 下次不同。
**D5 序列模式**（v3.0）：串起来看 N 步操作，而非逐条看。借鉴 Hermes 双重视角——正常视角看每步合理，模式视角看整体低效。

### Step 3 · 持久化改进

| 动作 | 条件 |
|------|------|
| ① 更新 MEMORY.md / 日志 | 新偏好或实质性工作未记录 |
| ② 更新 Skill 触发词 | 发现盲区 |
| ③ 添加踩坑经验 | 同一错误 ≥ 2 次 |
| ④ 创建 Skill 候选 | 任务含 ≥ 3 个可复用步骤 |

### Step 4 · 报告
本次改进 + 下次不同 + 建议。

### Step 5 · 质量评分
5 维度加权（目标达成 30% + 效率 25% + 工具选择 15% + 用户理解 15% + 知识复用 15%）→ 1.0-5.0 分。

> 完整评分维度表 + JSON 格式 + 趋势分析 → 见 references §2

## 🔄 自动触发

| 场景 | 阈值 | 执行 |
|------|------|------|
| 重试多次 | 同一错误 ≥ 2 次 | 记踩坑 |
| 用户不满 | 任何负面反馈 | D2 专项反思 |
| 复杂任务完成 | 调用 ≥ 5 次 | 静默自检 |
| 任务全部完成 | — | 完整反思 + 评分 |
| 新偏好 | 任何 | 立即写 MEMORY.md |

> 完整决策流程图 + 计数规则 → 见 references §4

## 📐 记忆写入规范

| 优先级 | 内容 | 置信度 |
|--------|------|--------|
| P0 | 用户说"记住" | [0.9] |
| P1 | 强调的选择 | [0.7] |
| P2 | 多次遵循（≥2）| [0.7] |
| P3 | 提及 1 次未强调 | [0.5] |
| P4 | AI 观察未声明 | [0.3] |

晋升/降级/冲突规则 → 见 references §3

## ⚙️ 核心规则

1. **诚实反思** — 不美化执行质量
2. **行动导向** — 必须产出可执行改进
3. **最小改动** — 只更新确定需要的
4. **用户透明** — 所有改动在报告中列出
5. **可撤销** — 追加不删除

## 🔗 联动

```
上游：所有 Skill 完成后强制触发
下游：skill-evolution(Skill更新) / skill-creator(新建)

关键路径：
执行完成 → 自检
  ├── 改进点 → 持久化
  ├── 可复用模式 → Skill 候选
  ├── 触发词盲区 → skill-evolution
  └── 踩坑 ≥ 2 次 → 自动追加
```

## 🔄 触发词自进化

用户表述未激活时：分析关键表述 → 抽象通用触发词 → 追加到 frontmatter triggers → 去重。

## 📚 踩坑经验

> AI 自动积累，请勿手动删除。格式：`- 场景：经验要点`

- 自动触发"建议触发"几乎从不触发 → 改为"必须触发"并明确阈值
