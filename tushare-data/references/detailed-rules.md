## Data Quality Rules（数据质量检查清单）

拉取完成后，**至少**执行以下检查：

| 检查项 | 方法 | 失败处理 |
|--------|------|---------|
| Schema 校验 | 确认返回字段符合预期 | 记录异常字段，继续处理 |
| 关键字段存在性 | open/high/low/close/volume 等 | 缺失则标记或补默认值 |
| 主键去重 | ts_code + trade_date 联合去重 | 保留最新记录 |
| 固定排序 | 按 trade_date 升序 | 重新排序 |
| 日期标准化 | 统一为 YYYYMMDD 格式 | 格式化转换 |
| 数值类型规范化 | 确保数值非字符串 | type conversion |

### 空结果诊断矩阵

空表 **不一定是失败**，按以下矩阵区分原因：

| 可能原因 | 判断方法 | 用户提示话术 |
|----------|---------|-------------|
| 非交易日 | 检查 trade_cal | "该日期为非交易日" |
| 区间无数据 | 标的上市时间 vs 请求区间 | "该标的在此期间无交易数据" |
| 股票未上市/已退市 | stock_basic 确认状态 | "该股票未上市或已退市" |
| 参数错误 | 回显参数请用户确认 | "请确认参数是否正确：{params}" |
| 接口权限不足 | Tushare 积分/权限检查 | "该接口需要更高积分权限" |

**不要把所有空结果都说成"接口坏了"。**

## Cache and Reuse Rules（缓存与复用策略）

### 缓存优先级

| 数据类型 | 缓存策略 | 更新频率 |
|----------|---------|---------|
| stock_basic | 全量缓存 | 每日增量更新 |
| trade_cal | 全量缓存 | 年度更新即可 |
| index_basic / index_classify | 全量缓存 | 低频变更 |
| 日线行情 | 按标的+日期缓存 | 增量追加 |
| 财报数据 | 按标的+报告期缓存 | 季度更新 |

### 文件命名规范

```
{接口名}_{ts_code}_{起始日}_{截止日}_{拉取时间}.扩展名
```

示例：
- `daily_600519.SH_20230101_20231231_20260427.csv`
- `fina_indicator_300750.SZ_20260427.parquet`

### 断点续跑

大任务支持断点：
1. 记录已完成分段到 `.progress` 文件
2. 重启时读取 progress 跳过已完成段
3. 最终合并所有分段 + 清理 progress

## Error Handling（错误分层处理）

### 分层原则

```
用户看到的（人话）           ← 一句话说明 + 可行建议
    ↓
调试层（按需展开）          ← 接口名 + 参数 + 异常原文
    ↓
系统层（日志中）            ← 完整 stack trace + 时间戳
```

### 用户可见层标准话术

| 错误类型 | 话术模板 |
|----------|---------|
| Token 未配置 | "TUSHARE_TOKEN 未设置。设置方式：`export TUSHARE_TOKEN=your_token`，注册地址：https://tushare.pro/register" |
| 权限/积分不足 | "`{接口}` 需要一定积分权限。当前积分可能不够，建议先用基础接口。" |
| 时间范围过大 | "请求范围 {n} 天，已自动改为分段拉取（每段 {step} 天）。" |
| 标的重名 | "'{name}' 匹配到多个标的：{候选列表}。请确认是哪一个？" |
| 结果为空 | "结果为空，可能原因：{诊断列表}" |

### 部分成功处理

如果部分分段失败：
1. ✅ 明确说："X 段成功，Y 段失败"
2. ✅ 交付已成功的部分数据
3. ✅ 列出失败段的错误信息
4. ❌ 不要说"成功完成"

## Natural Language Trigger Details（自然语言触发详解）

### 中文口语 → 任务映射

| 用户说法 | 解析为 | 默认行为 |
|----------|--------|---------|
| "最近" | 近 20 个交易日（行情）/ 近 3 个月（趋势） | 取 daily，截取尾部 |
| "财报" | 最近 8 个季度 + 最近年度 | income + fina_indicator |
| "强不强" | 走势涨跌 + 相对大盘强弱 + 成交活跃度 | daily + index_daily 对比 |
| "资金关注" | 净流入 + 活跃成交 + 龙虎榜/北向（按可用口径） | moneyflow 系列 |
| "估值" | PE(TTM) + PB + 股息率 + 历史分位 | daily_basic |
| "值得看" | 快速综合评估（行情+财务+估值+资金） | 综合研究简报流程 |

### 歧义处理优先级

当任务有多种合理解释时，按以下顺序选择：
1. 最常用/最直觉的解释（先跑这个）
2. 如果结果看起来不对，再做最小澄清
3. 不要在一开始就列 5 种可能问用户
