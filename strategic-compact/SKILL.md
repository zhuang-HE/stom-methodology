# Strategic Compact v2.0 — 主动式上下文管理

> **v2.0 核心变革**：不再等待上下文快爆才想起压缩。
> **每个任务开始时**自动启动预算评估和压力监控。
> 原文件 390行 → 精简至 ~200行，详细协议移至 `references/`。

---

## 🚀 主动模式：任务生命周期集成

### 任务启动时（必须做）

```
1. 快速分类任务规模：
   → 简单(1-3次调用) / 中等(4-8次) / 复杂(9-15次) / 超大型(16+)

2. 设定预算上限：
   → 简单: ~3K tokens   |  中等: ~10K tokens
   → 复杂: ~20K tokens  |  超大型: ~30K tokens

3. 记录起始基准线（工具调用计数 = 0）
```

### 任务执行中（持续监控）

```
每完成一次工具调用后，快速自检：

  调用次数 / 预期完成度 ≥ 0.7 ?  ──→ 🟢 继续
  
  调用次数 > 预算70% 且 完成度 < 50% ? ──→ 🟡 警告：
     1. 当前方法是否走偏？
     2. 有没有更直接的方法？
     3. 是否该换 Skill？

  同一子任务重试 ≥ 3 次？ ──→ 🔴 STOP，换路
```

### 阶段转换点（强制检查点）

```
研究 → 规划：✅ 推荐压缩（丢弃搜索过程，保留结论）
规划 → 实现：⚠️ 谨慎（保留路径+函数签名+变量名）
实现 → 测试：⚠️ 谨试（保留刚实现的代码上下文）
测试 → 交付：✅ 推荐（保留结果路径，丢弃调试过程）
子智能体返回后：✅ 推荐（摘要已返回，清理探索痕迹）
```

## 📡 压力信号与响应

| 级别 | 判定条件 | 自动响应 |
|------|---------|---------|
| 🟢 低 | 调用<5, 无Skill加载 | 正常执行 |
| 🟡 中 | 调用5-12, 1-2个Skill | 成本意识：精简输出格式 |
| 🟠 高 | 调用12-20, 2+Skill, 读过大文件 | 减压：暂停非核心分支→卸载子智能体→建议压缩 |
| 🔴 危险 | 调用>20, 连续读大文件, 对话很长 | 紧急：停探索性操作→只做必要修改→强烈建议/compact |

### 大文件保护

| 场景 | 规则 |
|------|------|
| SKILL.md > 400行 | 只读 frontmatter + 核心，其余按需 |
| 日志 > 200行 | 只读最近 50 行或搜索关键词 |
| Python > 300行 | search_content 定位，不全文读 |
| Markdown 报告 > 200行 | 只读目录 + 目标章节 |

### Skill 加载上限

| 已加载数 | 压力状态 | 能否再加载？ |
|----------|---------|-------------|
| 0-1 | 低-中 | ✅ 自由加载 |
| 2 | 中-高 | ⚠️ 注意预算 |
| 3+ | 🔴 高 | ❌ 不再主动加载，改用子智能体 |

> **硬规则**：已加载 2 个且压力为 🟠 时，不加载第三个。

## ✅ / ❌ 压缩时机速查

### 应该压缩（绿灯）

| 时机 | 保留 | 丢弃 |
|------|------|------|
| 研究→规划转换 | 结论、决策依据 | 搜索过程、中间读取内容 |
| 调试结束 | bug原因+修复方式 | 所有试错过程和错误日志 |
| 子智能体返回后 | 返回的摘要 | 子智能体内部探索痕迹 |
| 中间里程碑 | 已完成的结论+当前状态 | 已完成步骤的实现细节 |
| 同一文件被读≥3次 | 该文件的关键信息 | 重复读取的冗余内容 |

### 绝不压缩（红灯）

| 时机 | 原因 |
|------|------|
| **编码中途** | 变量名/函数签名/路径丢失 = 灾难 |
| **调试中途** | 推理链断裂 = 重新来过 |
| **子智能体刚返回未处理** | 结果丢失需重调 |
| **用户等关键决策** | 决策依据缺失 |
| **文件正在编辑** | 可能产生错误修改 |

## 📝 压缩指令模板

```markdown
# 好的压缩（带指令）
/compact 保留：1. 文件路径列表 2. 函数签名 3. 未完成TODO
              丢弃：已完成步骤细节、搜索过程

# 坏的压缩（空白！）
/compact          # ← 这会丢掉一切，绝对不要
```

### 分阶段指令示例

```
研究结束后 → /compact 保留：{技术选型结论、架构决策}
                      丢弃：所有搜索过程和中间read_file

实现中途被迫 → /compact 保留：{正在编辑的文件、函数签名、变量名}
                       丢弃：已完成的修改细节

调试结束后 → /compact 保留：{bug原因、修复方式}
                     丢弃：全部试错过程
```

## ⚙️ 五大铁律

1. **阶段优先于计数**：看任务阶段，不看纯 token 数
2. **实现中途零压缩**：宁可多读，不在编码中途丢上下文
3. **带指令压缩**：永远告诉系统"保留什么"，绝不空白压缩
4. **结论优于过程**：保留决策和结论，丢弃搜索和试错
5. **路径不可丢**：文件路径、函数签名、变量名是生命线

- 工具调用 ≥ 10 次
- 阶段转换点
- 子智能体返回后
- 长任务里程碑

### 下游（压缩后动作）
| Skill | 触发条件 |
|-------|---------|
| `workflow-loop` | 进入新的工作流闭环 |
| `self-improvement` | 记录本次压缩决策的经验 |
| `memory-consolidation` | 重要结论持久化到记忆文件 |

### 标准闭环
```
任务进行 → 监控(每步自检) → 检测触发条件 → 执行压缩 → 验证
                                                    ↓
                                              过重? → 回滚重压
                                                    ↓
                                              OK?  → 继续任务
```
