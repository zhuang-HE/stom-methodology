# Strategic Compact v2.1 — 主动式上下文管理

> **v2.0 核心变革**：不再等上下文快爆才压缩。**每个任务开始时**自动启动预算评估和压力监控。

> **详细协议**（Escalation / Completion Report / 三层检查）→ `references/detailed-protocols.md`

---

## 🚀 任务生命周期

### 启动时（必须做）

```
1. 分类任务规模：简单(1-3调用) / 中等(4-8) / 复杂(9-15) / 超大(16+)
2. 设预算：简单~3K / 中等~10K / 复杂~20K / 超大~30K tokens
3. 记录基准线（调用计数 = 0）
```

### 执行中（每步自检）

```
完成度 ≥ 70% ?    → 🟢 继续
调用 > 预算70% 且 完成度 < 50% ? → 🟡 警告：走偏了？换方法？换 Skill？
重试 ≥ 3 次 ?     → 🔴 STOP，换路
```

### 阶段转换点

```
研究→规划：✅ 丢弃搜索过程，保留结论
规划→实现：⚠️ 保留路径+函数签名+变量名
实现→测试：⚠️ 保留刚实现的代码
测试→交付：✅ 保留结果路径，丢弃调试过程
子智能体返回：✅ 摘要已返回，清理探索痕迹
```

## 📡 压力信号

| 级别 | 判定 | 响应 |
|------|------|------|
| 🟢 低 | 调用<5, 无Skill加载 | 正常 |
| 🟡 中 | 5-12调用, 1-2 Skill | 精简输出格式 |
| 🟠 高 | 12-20调用, 2+ Skill | 暂停非核心→卸载子智能体 |
| 🔴 危险 | >20调用, 长对话 | 停探索→只做必要→建议压缩 |

### Skill 加载上限

| 已加载数 | 能再加载？ |
|----------|-----------|
| 0-1 | ✅ 自由 |
| 2 | ⚠️ 注意预算 |
| 3+ | ❌ 改用子智能体 |

**硬规则**：已加载 2 个且 🟠 时，不加载第三个。

## ✅ / ❌ 压缩时机

### 应该压缩 ✅

| 时机 | 保留 | 丢弃 |
|------|------|------|
| 研究→规划 | 结论、决策依据 | 搜索过程 |
| 调试结束 | bug原因+修复 | 试错过程 |
| 子智能体返回 | 返回摘要 | 探索痕迹 |
| 同文件被读≥3次 | 关键信息 | 冗余 |

### 绝不压缩 ❌

- **编码中途**：变量名/函数签名/路径是生命线
- **调试中途**：推理链断裂 = 重来
- **子智能体刚返回未处理**
- **用户等关键决策**
- **文件正在编辑**

```
⚠️ 绝不要空白压缩：/compact ← 这会丢掉一切
```

## ⚙️ 五大铁律

1. **阶段优先于计数**：看任务阶段，不看纯 token 数
2. **实现中途零压缩**：宁可多读，不在编码中途丢上下文
3. **带指令压缩**：永远告诉系统"保留什么"
4. **结论优于过程**：保留决策和结论，丢弃搜索和试错
5. **路径不可丢**：文件路径、函数签名、变量名是生命线
